## Lane meanings

### Creative Hub

Current aggregation layer for Codrops originals and hand-picked external demos. Useful for runnable behavior and theme discovery. A Hub entry may point directly to Codrops `/Development` or `/Tutorials`, CodePen, Vercel, Netlify, or a contributor site. Source is optional.

Current curation headings include Featured Picks, Codrops Originals, Learn with Tutorials, GSAP Highlights, WebGL, Scroll-driven Animations, Grid-based Layouts & Effects, State-to-state Animations, Slideshows, Typography, Hover Interactions, and Cursor Fun.

### Webzibition

Visual exhibition of standout websites. Use for composition, typography, interaction, art direction, and reference discovery. Do not infer implementation, source availability, accessibility, or production quality from inclusion. At the 2026-07 check, individual WordPress Webzibition permalinks redirected to the Codrops home page; use the live archive/card as catalogue provenance and record its external website target separately.

### Tutorials

Step-by-step implementation articles. Strongest lane for article/demo/source triads, but still verify the linked repo, revision, license, browser behavior, and whether the code is tutorial-grade or production-ready.

### Playground

Experiments, concepts, and layouts that push interaction or motion ideas. Useful for bounded prototypes and visual vocabulary. Treat hard-coded geometry, assets, and environment assumptions as likely until source proves otherwise.

### Case Studies

Behind-the-scenes stories about intent, production constraints, architecture, motion, performance, and process. Strong contextual evidence; source code is not expected.

### Spotlights

Studio/Agency, Designer, and Developer stories. Use for practice, process, team, career, and craft lessons—not direct implementation recipes.

### Roundups

Motion Highlights, Websites Roundups, and Demos Roundups are curation evidence. They reveal clusters and vocabulary, but one roundup entry rarely proves a mechanism.

### CSS Reference, Blueprints, and Sketches

Historical/reference lanes. CSS Reference explains syntax/spec behavior; Blueprints provide older component concepts; Sketches isolate short visual experiments. Check age and browser/API drift.

### GitHub and embedded sandboxes

`github.com/codrops` is a source archive for many Codrops demos. Public post-content recounts found GitHub links in hundreds of posts and CodePen links in more than one hundred, with exact totals varying by parser breadth. Presence does not prove that a link is the official source for the article.

### Collective/newsletter

Curated design/development links. The WordPress archive reaches 879 records; later issues moved to Beehiiv and the newsletter changed from weekly to occasional updates in June 2025. Use as industry-reading history, not Codrops-authored implementation proof.

## Sampling guidance

For a representative study, stratify by:

- lane and editorial job
- publication era
- CSS/DOM/SVG/canvas/WebGL/WebGPU mechanism
- animation/rendering stack
- interaction job: hover, cursor, scroll, transition, navigation, gallery, typography, state change
- source availability and license
- experiment versus production framing

Anti-bias defaults:

- do not let one framework or current trend exceed one quarter of the consumed set unless the named theme requires it
- include historical and current items
- include non-demo editorial evidence
- include low-complexity alternatives
- include counterexamples and inaccessible/weak-evidence items
- reserve a 15–20 item holdout for a large method-building study
